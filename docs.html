Getting Started

AutoFapper is a powerful object-to-object mapping library that eliminates the tedious task of writing mapping code. With intelligent conventions and a fluent API, you can focus on what matters most.

Installation

Install AutoFapper via NuGet Package Manager:

Install-Package AutoFapper

Or using the .NET CLI:

dotnet add package AutoFapper

Basic Usage

Getting started with AutoFapper is incredibly simple:

// Configure your mappings
var configuration = new FapperConfiguration(cfg => {
    cfg.CreateMap<Source, Destination>();
});

// Create the fapper instance
var fapper = configuration.CreateFapper();

// Perform the mapping
var source = new Source { Name = "Test", Value = 42 };
var destination = fapper.Fap<Destination>(source);

Configuration

AutoFapper supports various configuration options to customize your mapping experience:

Custom Value Resolvers

cfg.CreateMap<Source, Destination>()
    .ForMember(dest => dest.FullName, 
        opt => opt.MapFrom(src => $"{src.FirstName} {src.LastName}"));

Ignore Properties

cfg.CreateMap<Source, Destination>()
    .ForMember(dest => dest.SecretProperty, opt => opt.Ignore());

Advanced Features

Nested Mappings

AutoFapper automatically handles nested object mappings:

cfg.CreateMap<Order, OrderDto>();
cfg.CreateMap<Customer, CustomerDto>();

// AutoFapper will automatically map nested Customer objects

Collection Mapping

Collections are handled seamlessly:

var sources = new List<Source> { /* ... */ };
var destinations = fapper.Fap<List<Destination>>(sources);

Conditional Mapping

cfg.CreateMap<Source, Destination>()
    .ForMember(dest => dest.Adult, 
        opt => opt.MapFrom(src => src.Age >= 18));

API Reference

IFapper Interface

The main interface for performing mappings:

  • TDestination Fap<TDestination>(object source) - Maps an object to the specified destination type
  • TDestination Fap<TSource, TDestination>(TSource source) - Strongly-typed mapping
  • void Fap<TSource, TDestination>(TSource source, TDestination destination) - Maps to an existing object

Configuration Methods

  • CreateMap<TSource, TDestination>() - Creates a mapping configuration
  • ForMember() - Configures individual member mappings
  • ReverseMap() - Creates a two-way mapping

Examples

DTO to Entity Mapping

public class UserDto
{
    public string Username { get; set; }
    public string Email { get; set; }
}

public class User
{
    public int Id { get; set; }
    public string Username { get; set; }
    public string Email { get; set; }
    public DateTime CreatedAt { get; set; }
}

// Configuration
cfg.CreateMap<UserDto, User>()
    .ForMember(dest => dest.CreatedAt, 
        opt => opt.MapFrom(src => DateTime.Now));

Flattening Complex Objects

cfg.CreateMap<Customer, CustomerViewModel>()
    .ForMember(dest => dest.AddressLine1, 
        opt => opt.MapFrom(src => src.Address.Line1))
    .ForMember(dest => dest.AddressCity, 
        opt => opt.MapFrom(src => src.Address.City));

Troubleshooting

Common Issues

Missing type map configuration

Ensure you've created a map for your types:

cfg.CreateMap<Source, Destination>();

Null reference exceptions

Use null checking in your custom resolvers:

cfg.CreateMap<Source, Destination>()
    .ForMember(dest => dest.Name, 
        opt => opt.MapFrom(src => src.Name ?? "Unknown"));

Performance issues

For high-performance scenarios, compile your mappings:

var fapper = configuration.CreateFapper();
fapper.CompileMappings();
